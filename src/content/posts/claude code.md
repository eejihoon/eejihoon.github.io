Cursor로 바이브 코딩하는 것을 팀에게 시연도 했으나 모두에게 아주 충격적인 경험은 아니었던 것 같다. 잠깐 놀라고, 다시 익숙한 작업 방식으로 돌아갔기 때문이다. 
나는 AWS 프로토타이핑 팀과 프로젝트를 진행하면서 Cursor를 결제해서 본격적으로 쓰기 시작했다. 파이썬을 전혀 몰랐지만 딸깍 하면 노트북 환경에서 Hugging Face 모델을 로딩하고, 임베딩하고, 오픈서치에서 검색하는 코드를 작성해줬다. 코드가 동작하지 않으면 에러와 함께 동작하지 않는다고 하면 커서가 알아서 고쳐줬다. 그때도 이 코드를 완벽하게 이해하지 못한다는 두려움이 있었다. 

그러다 클로드 코드라는 이름이 GeekNews 같은 데에서 슬슬 오르내리기 시작했다. 그 당시 준영님이 클로드 코드를 써보고 와서 팀에게 전파했다. 한 달 정도 뒤에는 우리 팀 모두가 클로드 코드를 사용해서 코딩하고 있었다. 그게 6월 즈음이었던 것 같다. 난 모든 코드를 클로드 코드로 작성하기 시작했다. 6월부터 지금까지 손으로 코딩한 양이 1만 라인도 안 될 것 같다. 회사 일을 2-3시간이면 쳐내고 내 작업을 할 수 있었다. 쉘위를 만들기 시작하면서 $100 플랜으로 업그레이드 했고, 그것도 부족해서 $200 가야 하나 고민하던 차에 8월부터였나 무신사에서 $200을 지급 받았고 그것마저 부족해질 때도 있었지만 대부분 괜찮았다. 무신사를 그만두고 팀오잉으로 오면서 MVP 제품 만들 때는 세션을 대여섯 개씩 돌리기도 했고, 9-10월즈음 앤트로픽이 주간 할당량까지 설정해둬서 $200플랜 + Cursor $60 플랜을 같이 썼다. 그즈음엔 자주 주간 제한에 걸렸는데, 또 몇 달 지나면서 사용량이 완화되었는지 리미트에 걸리는 일이 없어서 커서 구독은 그만뒀다. 서브 IDE로 커서를 사용할 때쯤 Cursor2.0이 발표되었는데 그때까지 바이브 코딩 경험 중 가장 좋았다.
